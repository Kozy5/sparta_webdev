- **데이터베이스(Database) 이무준** - **관계형 데이터베이스(Relational DataBase) 이무준** - **쿼리(Query) 이무준** - **API(Application Programming Interface) 이범근** - **오픈소스(Open Source) 이범근** - **SDK(Software Development Kit) 이지원** - **IDE(Integrated Development Environment) 이지원** - 1. **OS(Operating System)** 고진우 - 1-1 **OS(Operating System) 기본 개념** - **컴퓨터 시스템의 핵심 소프트웨어** 컴퓨터 하드웨어와 응용 프로그램 간의 상호작용(Interaction) 관리,제어 - **운영체제의 목적 →**  사용자와 하드웨어 사이의 인터페이스(Interface)를 제공 시스템 자원 효율적 관리 → 원활한 응용 프로그램 동작 지원,  **운영체제의 코어(핵심) →  커널(Kernel)** 일반적으로 커널 + 여러가지 기능(라이브러리, 시스템 프로그램 등등) == 운영체제(OS) - 1-2 **커널(Kernel)이란?** 사전 속 **Kernel** → **“알맹이, 핵심”**  **말 그대로 커널은 “운영체제의 핵심부 / 컴퓨터 자원 관리 역할** 수행 (컴퓨터 자원(System Resource) : CPU, 메모리, 파일, 네트워크, 입출력 장치 등) **커널은 항상 컴퓨터 자원만 바라봄 / 사용자와의 상호작용 지원 X** 사용자가 컴퓨터의 자원에 접근해야할때 → 시스템 프로그램 필요 대표적인 **시스템 프로그램 → 쉘(Shell) / 명령프롬프트(CMD)** 쉘(Shell) : 사용자 to 컴퓨터 명령 해석 프로그램. 즉, 커널과 사용자간의 다리 역할 **사용자는 쉘을 통해 커널에게 명령 가능** ### **컴퓨터의 세분화** 컴퓨터(Computer) 소프트웨어(Software) -> 응용 프로그램 (예 : MS Office, Chrome, VSCode 등) -> 운영 체제(OS, Operating System) (예 : Windows, MacOS, Linux 등) -> 시스템 프로그램 (예 : 쉘(Shell) 등) -> 커널(Kernel) : 운영체제의 핵심부 ### 커널의 추상화 - 물리적 자원 → 추상 자원 - CPU → Task - Memory → Page,Segment - Disk → File - Network 장치 → Socket - 외부장치(프린터,GPU 등)에 대한 접근 제공 ### 하드웨어(Hardware) (예 : 컴퓨터 본체, 모니터, 키보드 등) - 1-2 **OS(Operating System) 의 역할** **① 자원 관리** - 하드웨어 자원 효율적 관리해 여러 응용 프로그램이 동시에 실행될 수 있도록 한다. **② 메모리 관리** - 컴퓨터의 메모리를 효율적으로 관리해 응용 프로그램이 필요로하는 메모리 공간을 할당한다. (메모리의 할당과 해제, 가상 메모리 관리 등) **③ 프로세스 관리** - 실행 중인 응용 프로그램인 프로세스(Process)를 관리함 프로세스의 생성, 실행, 일시 정지, 시스템 중지 등 작업 제어 및 여러 프로세스 간의 자원 공유와 협력 관리 **④ 디스크 및 파일 시스템 관리** - 파일의 생성, 삭제, 복사, 이동 등의 작업을 처리 - 파일 시스템 = 데이터 영구적 저장과 접근을 제공하여 사용자 및 응용 프로그램이 데이터를 쉽게 관리 가능 **⑤ 입출력 관리 (**키보드, 마우스, 디스플레이, 프린터 등) - 입출력 장치와 상호작용을 관리 (사용자 and 응용 프로그램 and 입출력 장치 통신 지원 - 입출력 장치 성능 최적화 **⑥ 사용자 인터페이스** - 사용자와 컴퓨터의 상호작용을 위한 → 사용자 인터페이스 제공 텍스트 기반의 터미널 환경(CLI, Command Line Interface) 그래픽 사용자 환경(GUI, Graphical User Interface) 이런것들을 통해 사용자가 컴퓨터를 조작 가능(상호 작용) **⑦ 네트워킹** - 컴퓨터 네트워크 관리, 네트워크 프로토콜 지원 → 컴퓨터 간 통신 가능하게함 - 인터넷 연결, 데이터 전송, 네트워크 보안 등 관리 **⑧ 오류 처리** - 시스템 내, 외부의 오류와 예외 상황 처리함 - 프로세스의 비정상 종료, 메모리 오버플로우, 입출력 오류 등을 감지 및 처리 → 시스템의 안정성과 신뢰성을 유지한다. **⑨ 시스템 보안** - 시스템 보안 유지 목적으로 접근 제어, 사용자 인증, 암호화 등의 기능 제공 → 불법적인 접근, 데이터 유출 등으로부터 시스템 보호 / 안전한 환경 제공 - **2. 유닉스(Unix) 고진우** - 2-1 유닉스(**Unix**) **기본 개념** - 다중 사용자, 다중 작업(Multi-User, Multi-Tasking)을 지원하는 운영체제 - 초기에는 대형 컴퓨터에서 주로 사용 현재는 다양한 플랫폼에서 사용 - 명령 줄 인터페이스(command line interface, CLI)를 사용 (사용자는 셸과같은 프로그램을 이용해 명령어를 입력하여 컴퓨터를 제어) - 파일 시스템 중심 설계 ### 파일시스템이란? 최대한 간단한 개념으로 보자면 컴퓨터에서 파일이나 자료를 쉽게 발견 및 접근할 수 있도록 보관 또는 조직하는 체제 ### **UNIX파일시스템의 구조** UNIX파일 시스템의 구조는 디스크를 블록으로 분류하여 배치한 구조를 의미 부트블록, 슈퍼블록, I-node블록, 데이터블록으로 구성 **부트블록 :** 부팅시 필요한 코드 저장 블록 **슈퍼블록 :** 전체 파일 시스템 정보 저장 블록 **I-node 블록 :** 각 파일이나 디렉터리에 대한 모든 정보를 저장하고 있는 블록 **데이터 블록 :** 디렉터리 별로 디렉터리 엔트리와 실제 파일에 대한 데이터가 저장된 블록 또한 UNIX는 표준화된 인터페이스를 제공하여 여러 시스템 간의 호환성을 보장합니다. POSIX(portable Operating System Interface)는 UNIX와 유사한 운영체제간의 호환성을 위한 국제 표준 다양한 버전 존재 대표적으로 AT&T UNIX, BSD(Berkeley Software Distribution)UNIX, Solaris(Java), Linux 등이 있음 Linux는 UNIX와 유사한 오픈소스 운영체제이며 개방성과 유연성으로 인해 Linux는 현재 가장 많은 서버 시장을 차지중 - 2-2 유닉스(Unix)의 특징 UNIX는 주로 서버용 컴퓨터에서 사용되는 운영체제로 아래와 같은 특징이 있습니다. **1.** 시분할 시스템(Time Sharing System)을 위해 설계된 대화식 운영체제로, 소스가 공개된 개방형 시스템입니다. **2.** 대부분 C언어로 작성되어 있어 이식성이 높으며 장치, 프로세스 간의 호환성이 높습니다. **3.** 크기가 작고 이해하기가 쉽습니다. **4.** 다중 사용자, 다중 작업을 지원합니다. **5.** 많은 네트워킹 기능을 제공 → 통신망 관리용 운영체제로 적합 **6.** 트리구조의 파일 시스템을 가지고 있습니다. **7.** 디테일한 ****전문적인 프로그램 개발에 용이 **8.** 풍부한 애플리케이션 생태계 - **3. 리눅스(Linux) 고진우**
